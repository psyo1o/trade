run_bot.py
  - 국장(KR) / 미장(US) / 코인(COIN) 통합 매매 엔진.
  - 읽기/쓰기: config.json, bot_state.json, trade_history.json
  - manual_sell → _idempotent_*_sell(lane manual|phase5) → persist_idempotency → bal_read.invalidate
    → _apply_manual_sell_state_update(state=st) → ledger_apply.persist_* (부분/전량·통계·Layer2)
    → _run_manual_sell_position_sync (국·미·코인 조회 모두 성공 시 sync_all_positions)
  - 사이클: load_state 후 reconcile_positions_for_cycle(이번 cycle_tag filled 매도 ↔ positions)
  - 자동 매도 체결 후 ledger_apply.persist_position_set/remove (EXIT·HALF·Scale-Out·스윙 FULL 등)
  - persist_position_registration → ledger_apply (BUY 120초 멱등)
  - 핵심 함수: run_trading_bot() = 동기화 → 리스크 → 매도 → 매수
  - 앞단 단계화: _prepare_cycle_state(prune 멱등·bal_read.invalidate), _sync_positions_for_cycle,
    load_state·reconcile_positions_for_cycle, _build_market_context
  - build_account_snapshot_for_report: deps 잔고 = balance_read (fresh_balances·force_kis 시 refresh=True)
  - get_safe_balance("KR"|"US"|"COIN") → balance_read (레거시 시장 문자열 모드)
    (macro_snap: PCR·고래·환율 모멘텀·market_buy_allowed / market_buy_block_reason; macro_mult=1.0)
  - 매수 전 RS: _sort_buy_targets_by_rs → alpha_sizing. 비중: _position_ratio_with_vol_target
    (min(1/N, alpha_target_vol/ATR%), config alpha_target_vol 기본 0.02)
  - 본문: helper 2차 분리 + 시장별 3차 분리 완료 → execution/market_cycles/ (run_kr_cycle·run_us_cycle·run_coin_cycle).
  - run_trading_bot: TradingCycleContext → 위 3함수 → save_state (순서·태그 동일).
  - Phase5: 비중 판정 circuit_break.py · 루프 연동 execution/phase5_ops.py (run_bot 위임).
    합산 MDD는 account_circuit_use_total. docs/PHASE5_ACCOUNT_CIRCUIT.md
  - circuit_aux_last_* / phase5_share_anchor / phase5_pending_liquidation_markets
  - kis_balance_sync_mode on_trade: 평상시 장부+시세, 매매·강제새로고침·입출금 시 KIS 실조회
  - refresh_circuit_aux_from_brokers … adjust_capital·live sync 시
  - 관측성: [KR …], [US …], [COIN …] 태그. 매수 스캔은 rules.calculate_pro_signals 안에서 [V8],
    스윙 보조는 check_swing_entry 실패 시 [스윙] 한 줄(run_bot에서 출력).
  - 매수 시간창: 국 ``_is_kr_buy_window_now`` (KRX 15:30 KST 직전 N분), 미 ``_is_us_buy_window_now`` (NYSE 16:00 ET 직전 N분),
    코인 ``_is_coin_buy_window_now`` (일봉 KST 09:00 직전 N분, 업비트·바이낸스 공통). 창 안에서 V8→스윙·스캔·주문.
    코인 스캔 유니버스: 업비트 KRW·바이낸스 USDT **24h 거래대금 상위 N**(기본 **10**, ``config.json`` ``upbit_universe_top`` / ``binance_universe_top``).
  - Phase4 글로벌: 사이클 시작 ``🛡️ [Phase4 거시]`` / ``🛡️ [Phase4 글로벌]`` / ``🚫 [Phase4 글로벌] MK …`` 는 매 사이클.
    ``-> 🚨 국장|미장|코인 Phase4 글로벌 방어막`` 은 **해당 시장 매수 창 안**에서만(창 밖은 ``[KR|US|COIN 매수 대기]``만).
    Phase4 예산 축소(VIX/FGI) 제거 — ``macro_mult`` 항상 1.0. 시장별 글로벌만 ``-> 🚨 … 글로벌 방어막`` (매수 창 안).
  - 동기화: ``fetch_equity_held_lists_for_position_sync`` — 국·미 비정규장이면 KIS 보유 목록 조회 생략. ``sync_all_positions`` 에서 비장중 held 보강 등.
  - 일봉 OHLCV: ``get_cached_ohlcv`` — 메모리·``ohlcv_store`` 디스크(3일, ``d``+정렬+꼬리검증) → 국 KIS+pykrx 교차검증 → 미 KIS+yfinance 교차검증 → Stooq→yfinance 보강.
    목표 200봉; 텔레·GUI·매도선도 동일 일봉. ``OHLCV_MIN_BARS``; ``yfinance_guard`` 티커별 백오프.
  - 타임스탑 시계: ``_time_stop_hours_elapsed`` — COIN 연속; KR/US 거래일 24h 누적·주말·공휴 0 (XKRX/NYSE 캘린더).
    ``pandas_market_calendars`` break_start 공지 UserWarning — import 시 ``filterwarnings`` + ``is_yahoo_noise_line`` 텔레 제외.
  - 텔레 heartbeat: ``heartbeat_report`` → ``get_*_holdings_with_roi`` → ``_format_holding_line``
    (``전략:V8|스윙 | 매수가 | 현재가 | 최고가 | 매도선 | 보유기간``).
    ``_heartbeat_strategy_label`` — 장부 strategy_type·tier; 없으면 ``_strategy_from_trade_history_buy``(trade_history 최근 BUY reason).
    스윙 매도선은 ``_heartbeat_resolve_sl_p`` + ``get_swing_exit_display_price``.
  - 사이클 말: 매수 존에 들었으나 신규 체결 0건이면 텔레그램 ``[매수 패스]`` — 본문에 이번 존 시장만 ``KR`` / ``US`` / ``COIN`` 표기(복수 시 ``KR·US`` 등).

    바이낸스일 때 코인 칸·보유표·장부 탭 코인 가격은 USDT로 표시(스냅샷·서킷·Phase5는 내부 KRW 환산 유지).
    텔레 생존신고 코인 요약 USDT는 binance_display_cash_and_total_usdt()(원화 왕복 없음). TWAP 알림도 동일 규칙.
    잔고 API 실패 시 코인 상단 라벨은 bot_state last_coin_display_snapshot 폴백(성공 시 저장).
  - "예수금 새로고침" = 장부+시세([표시] 로그) / "KIS 강제 새로고침" = 실조회(🔁 로그, 스냅샷 저장). docs/KIS_GUI_DISPLAY.md
  - 스핀: 국·미·코인 max_positions + "설정 실시간 적용" → run_bot.MAX_* 즉시 + bot_state settings 저장.
  - 실시간 현황 보유표: **전략** 열(V8/스윙) 다음 **수동매도** 열 — 수량 QLineEdit(기본=행 보유 전량, 비우면 전량) + 매도 버튼 → _on_manual_sell_click
    (보유 초과·정수 주/코인 소수 검증) → run_bot.manual_sell.
  - 탭: 실시간 현황(7열: 시장·종목·수량·매수가·현재가·전략·수동매도), 매매 내역, 장부(텔레 동일 보유 한 줄+상세),
    매매·전략 안내(타임스탑·매도선·러너 5MA), 고점 보정(CapitalAdjustThread).
    실시간: 최고가·매도선·보유시간 없음 — 장부·heartbeat·15분 매매 로그에 ``format_holding_display_line`` 공용.
    보유 로그: 기동·잔고 갱신만으로 찍지 않음 — ``heartbeat_report``·매매 사이클 시.
    종목명 열 Interactive(약 절반 cap), 수량·가격·전략·매도 열 Stretch(_configure_holding_table).
    실시간·장부 수량: _build_live_qty_lookup 실계좌 우선. 코인 BUY qty는 체결 base 수량.

    GUI 타이포: 전역 15pt/15px (_dashboard_stylesheet + QFont 15). 로그·표·버튼·탭·상단 라벨 동일.

4) 한 사이클(run_trading_bot) 순서 (텍스트 흐름도)
--------------------------------------------------------------------------------

  1) _prepare_cycle_state … load·키 정규화·prune order_idempotency·bal_read.invalidate
  2) _sync_positions_for_cycle … sync_all_positions (API 전부 성공 시)
  3) load_state · reconcile_positions_for_cycle(이번 15분 filled 매도 ↔ positions)
  4) 리스크 … 시장별 MDD, Phase5 합산 서킷, Phase4 글로벌 알파 스냅샷, 섹터·(매수 전) AI(TREND_V8만)·Hurst 등
  5) 매도 … KR → US → COIN
       - 보유 로그: SWING_FIB → 매도선(스윙)=get_swing_exit_display_price, sl_p·max_p 매 사이클 갱신(기존 포지션 포함)
       - strategy_type==SWING_FIB → check_swing_exit(reference_price, trading_hours_held)
         HALF=50% 익절(≥1.5R) / FULL=피보·구름+시간가중·RSI — 1차익절은 매도선 아님
         HALF·FULL이면 continue, HOLD면 아래로 (V8 Scale-Out 스킵)
       - 분할익절(Scale-Out): strategy_type==TREND_V8 만 → 타임스탑 → SWING: _check_swing_trailing_exit
         (비러너 본절락 / 러너 5MA 이탈; FULL 5MA는 check_swing_exit 동시);
         피보/구름 FULL은 check_swing_exit, 손실구간 V8 hard_stop 스킵), TREND_V8: hard_stop·샹들리에
       - 타임스탑: V8 주식 72h/+4%, V8 코인 48h/+4%, SWING 주식 48h/+2%, SWING 코인 24h/+2%
         (KR/US: 거래일 24h 누적·휴장 0 — ``_time_stop_hours_elapsed``; COIN 연속)
       - 하드·매도선: V8 get_final_exit_price(샹들리에+20MA/ATR+본절락≥4%, 고정% 캡 없음).
         SWING get_swing_hard_stop_floor(피보·구름+영업24h후 시간가중 10%/24h)
       - V8 매수 초기 sl_p: calculate_pro_signals 기술 ATR선
         SWING 표시 get_swing_exit_display_price(하드+본절락; 러너+5MA)
       - 타임스탑 reason 태그: [V8_TIME_STOP_*], [SWING_TIME_STOP_*] — ticker_cooldowns는 guard.py
       - 타임스탑/하드스탑/샹들리에 전량 청산: 멱등 exit lane 단일 주문(♻️ 재사용).
         체결 후 ledger_apply.persist_position_remove. TWAP는 매수·Scale-Out 전용.
       - HALF/Scale-Out: lane filled 시 reconcile_ticker_lane 후 게이트; 체결 후 ledger_apply.persist_set
  6) 매수 … KR → US → COIN (시간창·필터 통과 시)
       - Phase4: market_buy_allowed false → 해당 시장 매수 창 전체 중단(V8·SWING 공통, US PCR 스윙 예외 없음)
       - BEAR 날씨: V8 신규 매수만 차단(_v8_trend_buy_allowed_in_weather), SWING_FIB 스윙 매수·스캔은 계속(Phase4 통과 시)
       - 후보 RS 정렬(alpha_sizing) → V8 calculate_pro_signals(Hurst<0.45) 우선, 실패 시 check_swing_entry
         (국장 갭+5%·KR/US/COIN reference_close 실시간 양봉; V8 필터 없음)
       - Portfolio Heat: 시장별 Σ(비중×ATR%) ≥ portfolio_heat_max_pct(기본 0.06) → 해당 시장 신규 매수 차단
       - Phase3 AI: _ai_false_breakout_buy_gate → strategy_type 전달·듀얼 프롬프트
       - 이미 보유(held_*) 종목은 신규 매수 스킵 (Holding Lock)
       - 비중: min(1/N, alpha_target_vol/ATR%) 후 예수금·최소주문 (macro_mult=1.0)
       - SWING 체결 후 register_swing_entry_risk_fields (1R·entry_initial_hard_floor)
  7) (선택) 매수 존·조건 충족인데 신규 체결 0건 → 텔레그램 [매수 패스]
  8) trade_history / bot_state 저장 (멱등·장부 변경 시 persist_idempotency / ledger_apply)
