1) 최상위 실행 파일 — 무엇을 켜야 하나

run_bot.py
  - 국장(KR) / 미장(US) / 코인(COIN) 통합 매매 엔진.
  - 읽기/쓰기: config.json, bot_state.json, trade_history.json
  - manual_sell → _apply_manual_sell_state_update(부분/전량 장부·통계·Layer2) → _run_manual_sell_position_sync
    (국·미·코인 조회 모두 성공 시 sync_all_positions)
  - 핵심 함수: run_trading_bot() = 동기화 → 리스크 → 매도 → 매수
  - 앞단 단계화: _prepare_cycle_state, _sync_positions_for_cycle, _build_market_context
    (macro_snap: PCR·고래·환율 모멘텀·market_buy_allowed / market_buy_block_reason; macro_mult=1.0)
  - 매수 전 RS: _sort_buy_targets_by_rs → alpha_sizing. 비중: _position_ratio_with_vol_target
    (min(1/N, alpha_target_vol/ATR%), config alpha_target_vol 기본 0.02)
  - 본문은 저위험 helper 추출로 2차 분리 완료. 시장별 통함수(_run_*_cycle) 3차 분리는 선택 과제.
  - Phase5: circuit_aux_last_* 갱신 + guard.apply_phase5_trailing_week_and_cooldown
    (서울 기준 월요일 주차 last_reset_week, peak_total_equity, 쿨다운 만료 후 고점 1회 리셋)
  - refresh_circuit_aux_from_brokers(state, path) … 브로커만으로 동일 키 재기록(adjust_capital 등)
  - 관측성: [KR …], [US …], [COIN …] 태그. 매수 스캔은 rules.calculate_pro_signals 안에서 [V8],
    스윙 보조는 check_swing_entry 실패 시 [스윙] 한 줄(run_bot에서 출력).
  - 매수 시간창: 국 ``_is_kr_buy_window_now`` (KRX 15:30 KST 직전 N분), 미 ``_is_us_buy_window_now`` (NYSE 16:00 ET 직전 N분),
    코인 ``_is_coin_buy_window_now`` (일봉 KST 09:00 직전 N분, 업비트·바이낸스 공통). 창 안에서 V8→스윙·스캔·주문.
  - Phase4 글로벌: 사이클 시작 ``🛡️ [Phase4 거시]`` / ``🛡️ [Phase4 글로벌]`` / ``🚫 [Phase4 글로벌] MK …`` 는 매 사이클.
    ``-> 🚨 국장|미장|코인 Phase4 글로벌 방어막`` 은 **해당 시장 매수 창 안**에서만(창 밖은 ``[KR|US|COIN 매수 대기]``만).
    Phase4 예산 축소(VIX/FGI) 제거 — ``macro_mult`` 항상 1.0. 시장별 글로벌만 ``-> 🚨 … 글로벌 방어막`` (매수 창 안).
  - 동기화: ``fetch_equity_held_lists_for_position_sync`` — 국·미 비정규장이면 KIS 보유 목록 조회 생략. ``sync_all_positions`` 에서 비장중 held 보강 등.
  - 사이클 말: 매수 존에 들었으나 신규 체결 0건이면 텔레그램 ``[매수 패스]`` — 본문에 이번 존 시장만 ``KR`` / ``US`` / ``COIN`` 표기(복수 시 ``KR·US`` 등).

run_gui.py  (일상 운영용 대시보드 — 화면 상세는 README.md 「GUI 사용 안내」)
  - PyQt5. import run_bot 순간 config 로드 → config 수정 후에는 GUI 재시작.
  - 상단: stats 라벨(약 3초 주기), 국/미/코인 예수·총평·보유수익률.
    코인 상단 ROI: _calc_coin_holdings_metrics — 바이낸스는 장부 buy_p·trade_history BUY 폴백.
    바이낸스일 때 코인 칸·보유표·장부 탭 코인 가격은 USDT로 표시(스냅샷·서킷·Phase5는 내부 KRW 환산 유지).
    텔레 생존신고 코인 요약 USDT는 binance_display_cash_and_total_usdt()(원화 왕복 없음). TWAP 알림도 동일 규칙.
    잔고 API 실패 시 코인 상단 라벨은 bot_state last_coin_display_snapshot 폴백(성공 시 저장).
  - "예수금 새로고침" = 일반 정책 조회 / "KIS 강제 새로고침" = KIS 1회 강제(자동 조회 최소 간격 약 25초).
  - 스핀: 국·미·코인 max_positions + "설정 실시간 적용" → run_bot.MAX_* 즉시 + bot_state settings 저장.
  - 실시간 현황 보유표 마지막 열: 수량 QLineEdit(기본=행 보유 전량, 비우면 전량) + 매도 버튼 → _on_manual_sell_click
    (보유 초과·정수 주/코인 소수 검증) → run_bot.manual_sell.
  - 탭: 실시간 현황(보유표·수동매도), 매매 내역(trade_history), 장부(positions, 수익률=max_p 기준, 마지막 열=전략명=tier),
    고점 보정(입출금, adjust_capital 동일·CapitalAdjustThread).
    실시간·장부 수량: _build_live_qty_lookup 실계좌 우선. 코인 BUY qty는 체결 base 수량.
    봇 브리핑 로그는 탭 아래 공통 영역(QSplitter, run_gui.py setSizes).
  - 매매: 기동 직후 즉시 1회 없음 → KST :00/:15/:30/:45 정렬 타이머만 (중복 실행 가드).
  - 기동 직후 약 150ms에 refresh_balance(sync_first=False) 1회로 화면만 선채움.
  - 매매 직전 refresh_balance로 max_p 맞춤 후 WorkerThread에서 엔진.
  - 텔레 heartbeat: 기동 직후 전송 없음 → KST :00/:30 슬롯 예약 후 해당 15분 매매 사이클 종료 뒤 전송 (백그라운드).
  - 스캐너: GUI 기동 시 start_scanner_scheduler 연동.
  - 네트워크 연속 실패 시 종료(BOT_DISABLE_NET_WATCH 로 감시 끔 가능).
  - 잔고 스레드: BalanceUpdaterThread + gui_table_adapter.build_rows_data.
  - 매매 스케줄: GUI 시작 즉시 1회 실행 없이 KST :00/:15/:30/:45 정렬 타이머에서만 run_trading_bot 호출.
  - US/KR 현재가: 장부 폴백(current=avg)·비장중 구간에서 정규화 후 yfinance 경로를 유도해 표시 안정화.

4) 한 사이클(run_trading_bot) 순서 (텍스트 흐름도)
--------------------------------------------------------------------------------

  1) 장부·주변 상태 준비 (_prepare_cycle_state 등)
  2) sync_all_positions … 실보유와 positions 정합
  3) 리스크 … 시장별 MDD, Phase5 합산 서킷, Phase4 글로벌 알파 스냅샷, 섹터·(매수 전) AI(TREND_V8만)·Hurst 등
  4) 매도 … KR → US → COIN
       - 보유 로그: SWING_FIB → 매도선(스윙)=get_swing_exit_display_price, sl_p·max_p 매 사이클 갱신(기존 포지션 포함)
       - strategy_type==SWING_FIB → check_swing_exit(reference_price=현재가)
         HALF=50% 익절(볼밴 상단·평단+2%↑) / FULL=피보·구름·RSI(평단↑) — 1차익절은 매도선 아님
         HALF·FULL이면 continue, HOLD면 아래로
       - 분할익절(Scale-Out) → 타임스탑·하드스탑 → SWING은 스윙 매도선 이탈(_check_swing_trailing_exit),
         TREND_V8만 샹들리에(check_pro_exit)
       - 타임스탑 reason 태그: [V8_TIME_STOP_*], [SWING_TIME_STOP_*] — ticker_cooldowns는 guard.py
       - 타임스탑/하드스탑/샹들리에 전량 청산: 단일 시장가 주문(재시도). TWAP는 매수·분할익절 전용(의도 설계)
  5) 매수 … KR → US → COIN (시간창·필터 통과 시)
       - Phase4 글로벌 차단은 시장별 **매수 창 안**에서만 ``-> 🚨 … 글로벌 방어막`` (사이클 시작 ``🚫 [Phase4 글로벌]`` 요약은 매 사이클)
