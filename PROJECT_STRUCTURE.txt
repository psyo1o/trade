프로젝트 구조/파일 설명 (2026-04-22 기준)
  — 모듈화 1~5 + 고도화 1번 + 관측성 + Phase5 월요일 주차 합산 MDD(peak_total_equity 등)
=====================================

일반 운영: run_bot.bat 또는 py -3.11 run_gui.py 로 GUI만 사용하면 됨.
run_bot.py 단독(헤드리스)은 선택 경로(콘솔·서버용).

1) 최상위 실행/설정 파일
----------------------
- run_bot.py
  - 국장/미장/코인 통합 자동매매 메인 엔진.
  - config.json, bot_state.json, trade_history.json을 읽고 쓰며,
    strategy/* + execution/* + api/* 모듈을 오케스트레이션.
  - 핵심 루프: run_trading_bot() (동기화 → 매도/리스크 → 신규매수).
  - last_kis_display_snapshot 저장/로드(save_last / load_last_…), 주말 KIS 스킵 시 GUI·텔레 국·미 숫자에 사용.
  - services/account_snapshot.py 공용 함수로 heartbeat/GUI 조회 규칙을 일원화(현재가·라벨 스냅샷).
  - run_trading_bot 앞단은 _prepare_cycle_state/_sync_positions_for_cycle/_build_market_context로 분리.
  - 고도화 1번 반영: run_trading_bot 본문 2차 분리(저위험 helper 추출 중심) 완료.
    보유시간 계산/로그, GUI 현재가 override, 수익률/하드스탑/타임스탑 조건, 코인 전처리 반복 블록 공용화.
  - 관측성: 국·미·코인 매수에서 예산/예수/정수주0/TWAP 미체결/BEAR·ADX 스킵 시 [KR|US|COIN …] 태그 로그.
  - 참고: KR/US/COIN 시장별 루프 통함수(_run_kr_cycle/_run_us_cycle/_run_coin_cycle) 3차 분리는 미적용(선택 과제).
  - Phase5: circuit_aux_last_* 갱신 + guard.apply_phase5_trailing_week_and_cooldown
    (월요일 서울 기준 주차 last_reset_week·peak_total_equity 앵커, 쿨다운 만료 후 고점 영점 리셋, 레거시 peak_equity_total_krw 미러).
    refresh_circuit_aux_from_brokers(state, path) — 브로커·업비트만으로 동일 키 재기록(adjust_capital.py 등).
  - heartbeat_report 생존신고·보유종목 줄·수동 매도 텔레: 장부 buy_time/buy_date 기준 보유 기간 접미사.

- adjust_capital.py
  - 수동 입·출금 시 peak_total_equity(및 peak_equity_total_krw 미러) 보정. 실행 시 refresh_circuit_aux_from_brokers 선행.
  - capital_adjustments 이력 append, bot_state.json 저장. config.json 필요.

- run_gui.py
  - PyQt5 기반 운영 GUI.
  - run_bot 모듈을 import해서 엔진/잔고/수동매도/로그를 UI로 제어.
  - 고점 보정(입출금) 탭: adjust_capital.py와 동일 로직으로 circuit_aux_* 갱신 후
    peak_total_equity 가감 및 capital_adjustments 기록(peak_equity_total_krw 동기).
  - KIS 잔고 조회 안정화: OPSQ0008/MCI 응답 시 백오프 재시도, 장외 스냅샷 우선,
    kr_bal/us_bal 기본 초기화로 UnboundLocalError 방지.
  - 자동 KIS 조회 최소 간격(약 25초) 보호 + 장외 추정치 유지, 필요 시 KIS 강제 새로고침 버튼으로 1회 호출.
  - 버튼 차이: "예수금 새로고침"은 일반 정책(장중/쿨다운) 준수, "KIS 강제 새로고침"은 정책을 우회해 KIS 1회 강제 호출.
  - BalanceUpdaterThread는 공용 build_account_snapshot_for_report를 호출하고,
    GUI 정책(allow_kis_fetch/with_backoff)을 콜백으로 주입해 기존 동작(백오프/장외 스냅샷/강제조회)을 유지.
  - 매매는 KST 분기(:00/:15/:30/:45); 텔레그램 생존보고는 기동 즉시 1회 후 KST :00/:30 정렬.
  - KIS 주말 점검 창: 국·미 잔고 API 미호출, last_kis_display_snapshot·장부로 표시.
    보유수량은 positions[티커].qty(평일 동기화/매수 시 기록), 없으면 표시 1 폴백.
  - 설정(config.json)은 run_bot import 시 로드되므로 변경 후 GUI 재시작 필요.

- screener.py
  - 한국투자 API 기반 국장 스크리너.
  - HTS 조건검색 결과를 kr_targets.json에 저장.
  - 스케줄러(run_bot.start_scanner_scheduler)에서 매 거래일 14:50 KST 실행
    (국장 매수 가능 구간 직전 갱신 목적).

- us_screener.py
  - 미장 감시 유니버스 재빌드(run_bot.get_top_market_cap_tickers(force_refresh=True)).
  - us_universe_cache.json 갱신. 스케줄러에서 매 거래일 15:20 US/Eastern 실행
    (미장 매수 창 15:30 ET 기준 약 10분 전, DST 자동).

- config.json
  - API 키, 텔레그램, test_mode, TWAP, 거시필터 등 런타임 설정.

- bot_state.json
  - 장부/상태 파일.
  - positions — 티커별 buy_p, sl_p, tier, qty, strategy_type, entry_fib_level 등.
    qty는 평일 sync_positions·매수 체결 시 갱신(주말 GUI 수량 표시).
    strategy_type은 TREND_V8/SWING_FIB 매도 분기, entry_fib_level은 스윙 하드스탑 기준.
  - cooldown(단기), ticker_cooldowns(매도 후 재진입 쿨다운 ISO 시각), stats.
  - Phase5: peak_total_equity, last_reset_week(예 2026-W16), account_circuit_peak_reset_pending,
    account_circuit_cooldown_until, peak_equity_total_krw(미러),
    circuit_aux_last_kr_krw, circuit_aux_last_usd_total(미장 USD 합산), circuit_aux_last_coin_krw.
  - capital_adjustments — adjust_capital.py 입출금·고점 보정 로그(선택).
  - last_kis_display_snapshot — 국·미 예수·총평·수익률 직전 성공값(saved_at 포함). 주말 GUI/텔레 표시용.

- us_universe_cache.json
  - 미장 S&P500 시총 상위 100 + Nasdaq100 중 Tier1 제외 상위 50(최대 150) 및 GICS 섹터.
  - 24시간 TTL; us_screener가 강제 재빌드 시에도 갱신.

- trade_history.json
  - 매매 이벤트 append 기록.


2) 디렉터리별 역할
------------------
- api/
  - 외부 브로커/데이터 API 래퍼 레이어.
  - kis_api.py: 한국투자 토큰/브로커/잔고/주문.
  - upbit_api.py: 업비트 잔고/주문.
  - macro_data.py: VIX, Fear&Greed 원시 지표 수집.
  - kis_parsers.py: KIS output2(list/dict) 혼합 파싱 공통 함수(예수금/총평가/외화예수금 fallback/수량 추출).

- strategy/
  - 매수/매도 판단 로직(규칙·필터).
  - rules.py: V5 전략 코어(OHLCV, 시그널, 청산 가격). yfinance 재시도; 국내 일봉은 KIS 우선 가능.
  - indicators.py: V8.0 지표 유틸(get_safe_atr). 결측치/0종가/이상 TR/최소 거래일 검증 후 ATR(14) 계산, 실패 원인 로그 출력.
  - sector_lock.py: 동일 GICS 섹터 과집중 제한
    (한도 = max(1, (max_positions_XX + 1) // 2) 형태로 올림).
  - ai_filter.py: false-breakout(휩쏘) 필터.
  - macro_guard.py: 거시 방어막 정책(차단/축소/정상).

- execution/
  - 실행/리스크/동기화 유틸.
  - guard.py: 장부 I/O, cooldown / ticker_cooldowns, MDD, 합산 서킷(월요일 주차 MDD·쿨다운 후 고점 리셋), can_open_new(시장별 슬롯, 코어 예외 없음);
    load_state 시 positions.scale_out_done 기본값 보강; 매도 사유별 ticker_cooldowns 설정·조회;
    기본 장부에 last_kis_display_snapshot 키 병합.
  - sync_positions.py: 실계좌↔장부 자동복구/유령정리/평단보정·보유수량(qty) 동기화(평일 API).
    주말 KIS 점검 창에서는 국·미 실계좌 시드만 건너뛰고 코인 시드는 유지(로그 한 줄).
  - order_twap.py: 분할 매수/매도(TWAP) 슬라이스 계획/실행.
  - scale_out.py: V8.0 동적 50% 분할 익절(Scale-Out) — entry_atr*3.0 목표(부재 시 +30% fallback),
    조건·수량·최소명목·장부 보정·TWAP 연동 헬퍼.
  - circuit_break.py: 합산 자산 drawdown 판정 유틸.

- services/
  - account_read_facade.py: KR/US 보유 조회 및 detail/info 공통화(장외/실패 폴백 포함).
    주말·점검·output1 비어 있음·예외 시 장부 폴백 전에 ⚠️/📌 로그(빈 반환만 하지 않음).
  - account_snapshot.py: GUI/heartbeat 공용 계좌 스냅샷/현재가 해석 서비스.
    - resolve_display_current_price: KR(API 우선), US(API→yfinance→OHLCV), COIN(pyupbit) 규칙 통일.
    - build_account_snapshot_for_report: KR/US/COIN 라벨 + holdings + 원본 balances를 한 번에 구성.
      주말·allow_kis_fetch 생략·KR/US 조회 예외·스냅샷 저장 실패 시 snapshot 태그 로그·직전 라벨 폴백.
  - gui_table_adapter.py: GUI 실시간 현황 테이블 행 조립 분리.

- utils/
  - 공통 보조 모듈.
  - logger.py: 자정 롤오버(30일) 로깅 + stdout/stderr 리다이렉트.
  - telegram.py: 알림 및 atexit 종료 핸들러.
  - helpers.py: 티커 정규화, 이름 조회, JSON 입출력, 코인 최소수량 기준,

- tests/

  - test_kis_parsers.py: KIS 파서 최소 회귀 테스트.

    run_bot(refresh_circuit_aux_from_brokers, circuit_aux_*), 루트 adjust_capital.py(고점 수동 보정)
  - 매도 후 재진입: ticker_cooldowns (guard.py, Phase 번호 없음)

- V7.1 액티브 전용: 코어 자산 예외 없음. 모든 보유 종목 동일 매도·샹들리에·슬롯 규칙.

  - bot_state positions[*].scale_out_done: 기본 false; true = 해당 포지션에서 분할 익절 1회 이미 완료(재시도 없음).
  - 트리거: 현재가 >= 매수가 + (entry_atr*3.0), entry_atr 부재/0이면 +30% fallback. 명목>=300만원(KRW).
  - 국·미: 수량 // 2 (1주면 스킵); 코인: 0.5× 후 truncate. TWAP: config twap_* / run_bot TWAP_ENABLED.
- V8.0 ATR 센서/저장: strategy/indicators.py(get_safe_atr) + run_bot 보유/진입 루프.
  - positions[*].entry_atr(진입 박제), positions[*].current_atr(실시간) 저장.
  - 검증 실패 시 경고 로그 + None(봇 중단 없음), 매도 엔진은 fallback 값으로 안전 동작.
  - 샹들리에: max_price - current_atr*2.5 (부재 시 현재가*2% fallback).
  - 초기 하드스탑 듀얼: ma20-ATR*1.0 vs 현재가-ATR*2.0 (ATR 부재 시 -10% fallback).
- 멀티 전략(최소 통합): run_bot 진입은 V8 우선, 미충족 시 check_swing_entry 보조 평가.
  - 스윙 체결 장부: strategy_type="SWING_FIB", entry_fib_level 저장.
  - 스윙 포지션 매도: check_swing_exit 전용(FULL/HALF/HOLD)으로 처리하고 V8 매도 경로는 타지 않음.
  - 비스윙 포지션은 기존 V8 매도 루프(분할익절/타임스탑/하드스탑/샹들리에) 유지.
- 코인 먼지 수량 기준: utils/helpers.py
  - COIN_MIN_POSITION_QTY (= 0.0001)
  - coin_qty_counts_for_position(qty)로 실질 보유 여부 통일 판정.


4) 동작 흐름(요약)
-----------------
1. run_bot 시작 → config 로드, 브로커/텔레그램/로거 준비.
2. run_trading_bot() 주기 실행.
3. sync_all_positions()로 실보유와 장부 정합.
4. 리스크 체크(MDD/합산서킷/거시/섹터/AI필터).
5. 매도: V8.0 동적 분할 익절(조건 시) → 타임스탑·하드스탑·샹들리에 등. 이후 신규 매수(시장·시간·슬롯 조건 충족 시).
6. 체결/상태를 trade_history.json, bot_state.json에 반영.

참고 — 미장 유니버스: run_bot.get_top_market_cap_tickers (S&P Top100 + NDX 비중복 50,
  최대 150). 국장 스캐너 14:50 KST / 미장 us_screener 15:20 ET.
  OHLCV는 국내 6자리 KIS 일봉 우선·yfinance 보완.
  매도 후 재진입은 ticker_cooldowns.
  증권사 주말 점검(KIS): helpers.kis_equities_weekend_suppress_window_kst 참고 — 코인(업비트)만 실시간.
  국·미 예수·총평 등은 last_kis_display_snapshot(평일 마지막 성공 조회). 보유종목 수량은 positions.qty.
  업비트 매수: 가용 KRW×0.999 캡·최소 5000원(run_bot 상수).
  Phase5 합산 MDD: peak_total_equity + last_reset_week + circuit_aux_* (원화 합산 시 환율 estimate_usdkrw).
  수동 입출금 고점 보정: py -3.11 adjust_capital.py (config.json 필요).
