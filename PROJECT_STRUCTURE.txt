1) 최상위 실행 파일 — 무엇을 켜야 하나
--------------------------------------------------------------------------------

run_bot.py
  - 국장(KR) / 미장(US) / 코인(COIN) 통합 매매 엔진.
  - 읽기/쓰기: config.json, bot_state.json, trade_history.json
  - manual_sell → _apply_manual_sell_state_update(부분/전량 장부·통계·Layer2) → _run_manual_sell_position_sync
    (국·미·코인 조회 모두 성공 시 sync_all_positions)
  - 핵심 함수: run_trading_bot() = 동기화 → 리스크 → 매도 → 매수
  - 앞단 단계화: _prepare_cycle_state, _sync_positions_for_cycle, _build_market_context
  - 본문은 저위험 helper 추출로 2차 분리 완료. 시장별 통함수(_run_*_cycle) 3차 분리는 선택 과제.
  - Phase5: circuit_aux_last_* 갱신 + guard.apply_phase5_trailing_week_and_cooldown
    (서울 기준 월요일 주차 last_reset_week, peak_total_equity, 쿨다운 만료 후 고점 1회 리셋)
  - refresh_circuit_aux_from_brokers(state, path) … 브로커만으로 동일 키 재기록(adjust_capital 등)
  - 관측성: [KR …], [US …], [COIN …] 태그. 매수 스캔은 rules.calculate_pro_signals 안에서 [V8],
    스윙 보조는 check_swing_entry 실패 시 [스윙] 한 줄(run_bot에서 출력).

run_gui.py  (일상 운영용 대시보드 — 화면 상세는 README.md 「GUI 사용 안내」)
  - PyQt5. import run_bot 순간 config 로드 → config 수정 후에는 GUI 재시작.
  - 상단: stats 라벨(약 3초 주기), 국/미/코인 예수·총평·보유수익률.
  - "예수금 새로고침" = 일반 정책 조회 / "KIS 강제 새로고침" = KIS 1회 강제(자동 조회 최소 간격 약 25초).
  - 스핀: 국·미·코인 max_positions + "설정 실시간 적용" → run_bot.MAX_* 즉시 + bot_state settings 저장.
  - 실시간 현황 보유표 마지막 열: 수량 QLineEdit(기본=행 보유 전량, 비우면 전량) + 매도 버튼 → _on_manual_sell_click
    (보유 초과·정수 주/코인 소수 검증) → run_bot.manual_sell.
  - 탭: 실시간 현황(보유표·수동매도·로그), 매매 내역(trade_history), 장부(positions, 마지막 열은 tier),
    고점 보정(입출금, adjust_capital 동일·CapitalAdjustThread).
  - 매매: 기동 약 1초 후 run_trading_bot 1회 → 이후 KST :00/:15/:30/:45 (중복 실행 가드).
  - 매매 직전 refresh_balance로 max_p 맞춤 후 WorkerThread에서 엔진.
  - 텔레 heartbeat: 기동 1회 + KST :00/:30 (백그라운드).
  - 스캐너: GUI 기동 시 start_scanner_scheduler 연동.
  - 네트워크 연속 실패 시 종료(BOT_DISABLE_NET_WATCH 로 감시 끔 가능).
  - 잔고 스레드: BalanceUpdaterThread + gui_table_adapter.build_rows_data.
  - 매매 스케줄: GUI 시작 즉시 1회 실행 없이 KST :00/:15/:30/:45 정렬 타이머에서만 run_trading_bot 호출.
  - US/KR 현재가: 장부 폴백(current=avg)·비장중 구간에서 정규화 후 yfinance 경로를 유도해 표시 안정화.

screener.py
  - 국장 HTS 조건검색 결과 합쳐 kr_targets.json 저장 (로컬만, Git 무시).

us_screener.py
  - 미장 유니버스 재빌드 → us_universe_cache.json. 스케줄 15:20 US/Eastern.

adjust_capital.py
  - 수동 입출금 시 peak_total_equity 단일 키 보정.
  - 실행 시 refresh_circuit_aux_from_brokers 선행. capital_adjustments 기록.

4) 한 사이클(run_trading_bot) 순서 (텍스트 흐름도)
--------------------------------------------------------------------------------

  1) 장부·주변 상태 준비 (_prepare_cycle_state 등)
  2) sync_all_positions … 실보유와 positions 정합
  3) 리스크 … 시장별 MDD, Phase5 합산 서킷, 거시·섹터·(매수 전) AI 등
  4) 매도 … KR → US → COIN
       - strategy_type==SWING_FIB → check_swing_exit (HALF/FULL이면 처리 후 continue, HOLD면 아래로)
       - 분할익절(Scale-Out, TWAP 임계 적용) → 타임스탑·하드스탑·샹들리에
       - 타임스탑 reason 태그: [V8_TIME_STOP_*], [SWING_TIME_STOP_*] — ticker_cooldowns는 guard.py
       - 타임스탑/하드스탑/샹들리에 전량 청산: 단일 시장가 주문(재시도). TWAP는 매수·분할익절 전용(의도 설계)
  5) 매수 … KR → US → COIN (시간창·슬롯·필터 통과 시)
       - V8 calculate_pro_signals 우선, 실패 시 check_swing_entry 보조
  6) trade_history / bot_state 저장
